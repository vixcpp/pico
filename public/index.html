What Pico checks

Runtime features connected in one app

HTTP routes

Pico exposes health, status, KV, event, job, and WebSocket diagnostic routes through the Vix application router.

Static files

This page is served from the public directory through the Vix static file system.

Durable KV

Runtime counters such as boot count, HTTP requests, WebSocket messages, and job runs are persisted through Vix KV.

SQLite events

Runtime events are stored in SQLite so Pico can observe what happened during requests, jobs, and WebSocket activity.

ThreadPool jobs

Pico submits diagnostic work to the Vix ThreadPool and exposes worker metrics through the status API.

WebSocket runtime

Pico prepares a WebSocket server and lets Vix AttachedRuntime run HTTP and WebSocket together under one lifecycle.
